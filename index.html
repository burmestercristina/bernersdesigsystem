🔴 Prioritarios — empezar aquí

01
Badge / Status pill
El estado de campaña (activo, pausado, borrador, error) aparece en todas las vistas. Sin este componente cada pantalla lo resuelve diferente.
02
Skeleton / Loading state
Momento de máxima ansiedad del usuario — sin datos visibles. Imprescindible en un producto con dashboards y métricas en tiempo real.
03
Empty state
Sin campañas, sin datos, primer uso. Un empty state bien diseñado guía al usuario hacia la acción correcta. Sin él, la app parece rota.
04
Stepper / Steps
El flujo de creación de campaña tiene varios pasos diferenciados. Sin un stepper claro el usuario pierde el contexto de dónde está.
05
Stat / KPI card
Número grande + tendencia + delta porcentual. Componente nuclear para dashboards de rendimiento de campañas.
06
Charts (line, bar, donut)
Visualización de métricas de campañas. Definir los tokens de color y estilo para que todas las gráficas sean coherentes entre sí.
07
Top bar / Header de app
Con avatar, notificaciones y contexto de cuenta/marca. Marco estructural de la app que afecta a todas las pantallas.
08
Input numérico / moneda
Crítico para la entrada de presupuestos de campaña. Necesita manejo específico: formato, límites, validación inline.
